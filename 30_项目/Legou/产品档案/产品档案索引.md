# Legou 产品档案索引

## 用途

这里沉淀产品核心内容，用于后续发行、买量、创意、数据分析和产品反哺决策。产品档案不是宣传稿，而是“可调用的产品事实与判断库”。

使用原则：

- 稳定事实和阶段判断分开写。
- 数据结论必须标注来源和证据强度。
- 产品档案必须包含核心数据快照，不能只写定位和卖点。
- 创意方向必须服务产品真实卖点和当前生命周期。
- 被新数据推翻的判断要标记废弃，不直接覆盖历史。

## 通用调用方式

做任何 ROK / COD 相关判断前，先读取对应产品档案，再读取相关数据分析或素材资料。

建议问题模板：

```text
当前目标是什么：拉新 / 回流 / 提收入 / 测卖点 / 验证渠道？
这个动作服务规模还是单位价值？
素材吸引的人，产品是否接得住？
当前判断基于哪些证据？
缺什么数据？
最小验证动作是什么？
决策阈值是什么？
```

## 数据字段要求

每个核心产品档案至少保留：

- 当前截面：新增、DAU、充值设备数、流水、付费率、ARPU、ARPPU。
- 变化趋势：环比、同比、全周期峰值/回落/稳定状态。
- 用户结构：自然 vs 广告、iOS vs AOS。
- 渠道结构：TOP 渠道和应用市场。
- 决策含义：这些数据说明当前应该做什么、不该做什么。
- 待核验项：截图无法解释的原因和需要补充的数据。
