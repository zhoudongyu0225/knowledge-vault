## 可复用观察框架

每次收录一个玩法，不按“像不像某爆款”记录，统一拆成：

| 字段 | 要回答的问题 |
|---|---|
| 基础动作 | 玩家每 1-3 秒在做什么？ |
| 状态载体 | 方块、传送带、弹球、箭头、容器或角色如何承载状态？ |
| 信息可见性 | 下一步、目标、危险和奖励是否一眼可见？ |
| 压力来源 | 时间、空间、顺序、随机、资源还是失败惩罚？ |
| 反馈 | 合成、碰撞、清空、倍增、成长是否足够清楚和爽快？ |
| 长线承接 | 关卡、装修、订单、活动、收集或社交如何接住核心玩法？ |
| 商业化位置 | 广告或付费是否顺着玩家目标出现，而不是强行打断？ |
| 差异化 | 新增机制真正改变了决策，还是只换了表现？ |
| 制作风险 | 美术量、关卡量、物理稳定性、数值调优和内容消耗是否可控？ |

## 当前最值得沉淀的判断

- 传送带的价值不只是“最近流行”，而是把状态变化、来料、顺序和循环同时做成可视反馈。
- 休闲融合的上限取决于“多一层目标但不多一层理解成本”。机制越多，越要保住第一眼的核心动作。
- 题材有价值的前提是它能解释机制。例如沙子天然适合流动，食物天然适合加工和订单，不能只换美术皮。
- 随机与赌博感可以提高情绪强度，但必须让玩家保留可理解的选择，否则容易从刺激变成不公平。
- 玩法观察只证明“值得做原型”，不能证明“值得立项”；立项还需成本、用户、留存、商业化和供量证据。
