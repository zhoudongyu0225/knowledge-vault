## 可复用观察框架

每次收录一个玩法，不按“像不像某爆款”记录，统一拆成：

| 字段 | 要回答的问题 |
|---|---|
| 基础动作 | 玩家每 1-3 秒在做什么？ |
| 状态载体 | 方块、传送带、弹球、箭头、容器或角色如何承载状态？ |
| 信息可见性 | 下一步、目标、危险和奖励是否一眼可见？ |
| 压力来源 | 时间、空间、顺序、随机、资源还是失败惩罚？ |
| 反馈 | 合成、碰撞、清空、倍增、成长是否足够清楚和爽快？ |
| 长线承接 | 关卡、装修、订单、活动、收集或社交如何接住核心玩法？ |
| 商业化位置 | 广告或付费是否顺着玩家目标出现，而不是强行打断？ |
| 差异化 | 新增机制真正改变了决策，还是只换了表现？ |
| 制作风险 | 美术量、关卡量、物理稳定性、数值调优和内容消耗是否可控？ |

## 当前最值得沉淀的判断

- 传送带的价值不只是“最近流行”，而是把状态变化、来料、顺序和循环同时做成可视反馈。
- 休闲融合的上限取决于“多一层目标但不多一层理解成本”。机制越多，越要保住第一眼的核心动作。
- 题材有价值的前提是它能解释机制。例如沙子天然适合流动，食物天然适合加工和订单，不能只换美术皮。
- 随机与赌博感可以提高情绪强度，但必须让玩家保留可理解的选择，否则容易从刺激变成不公平。
- 玩法观察只证明“值得做原型”，不能证明“值得立项”；立项还需成本、用户、留存、商业化和供量证据。

## 2026-07-13 新文分析：《Knock Out》如何改造《Smash Fest》

文章不是在讲“换一套美术继续抄”，而是在展示同一成熟原型可以从四层同时微调：

1. **参数层**：缩小球体、降低单发威力，提高瞄准要求和多次击打概率，让道具有了真实需求。
2. **关卡层**：把击打次数视作步数，逐步加入木头、果酱、石块、冰块、移动台面和遮挡板，让物理直觉承载类似三消的障碍学习。
3. **经济与活动层**：第 9/13/17/20 关分批开放道具，第 20/35/71 关开放连胜、40 人异步赛和 3 人竞速，避免只验证前几十关的核心动作。
4. **品牌层**：用塑料玩具材质、Busters 共享角色和统一通关表现，把单款产品的识别积累到系列厂牌。

可转成三个原型检查项：

- 不只调规则，也记录弹体大小、速度、威力、容错等参数怎样改变难度与道具消耗。
- 原型评审至少模拟“核心动作—障碍—道具—活动”的分层开放，不能只看首局爽感。
- 如果计划做产品矩阵，统一角色、材质和情绪语言应在首款产品阶段定义，而不是成功后再贴 IP。

证据边界：正文没有独立留存、收入或投放数据，以上只能形成实验假设，不能证明《Knock Out》已经商业成功，也不能证明该路线适合当前团队。
