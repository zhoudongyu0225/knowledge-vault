本文件用于让新的 Codex 任务快速恢复工作状态，不代替产品档案和原始分析。开始续作时先读本文件，再按关联链接读取具体资料。

新任务可直接说：

## 当前关键判断

- ROK：成熟长线经营盘。7 月经营收入节奏较强，但新增付费率、付费新增 D7 和新付费成本显示新增质量继续变弱。
- COD：2024 年证明过高质量和纯投放回本能力；2025 年后模型退化。2026 年 7 月活跃、留存、付费率和收入效率同时下降，需要优先排查活动、版本、市场和流量结构。
- Beagle：早期验证项目。当前能低成本获量，但 D1 到 D7 留存断层、复购和直接回收尚未成立，不适合规模化投放。

## 跨设备边界

- Git 会同步正式文档、索引、方法和本文件。
- Codex 本地任务记录和本地 memories 不作为跨设备可靠上下文；长期规则必须写入 `AGENTS.md` 或知识库文档。
- `90_附件/私有资料/` 下的 DAP 原始敏感数据被 Git 排除，家里电脑默认只能看到整理后的分析，不能直接重新计算原始数据。
- 需要原始敏感数据跨设备使用时，应另行采用加密存储，不得直接加入当前 Git 仓库。

## 家里电脑继续工作的最短路径

1. 克隆或拉取 `knowledge-vault`。
2. 在 Codex 中打开 `knowledge-vault` 本地目录。
3. 新建任务并使用上面的恢复上下文提示。
4. Codex 自动读取根目录 `AGENTS.md`，再根据本文件加载相关产品资料。
5. 开始工作前先执行 `git pull --ff-only`；结束后由每日同步任务或人工提交推送。
