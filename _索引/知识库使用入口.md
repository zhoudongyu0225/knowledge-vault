# 知识库使用入口

这个入口不是文件夹目录，而是问题目录。以后优先按“我要解决什么问题”来调用知识库，不要求记住文件位置。

## 直接发给我时

你可以直接把文章、聊天记录、截图文字、观点或数据发到对话里，然后加一句指令：

- `整理进知识库`
- `先存，不用深度分析`
- `和游戏发行有关，归档一下`
- `以后做投放/素材/产品决策可能有用`
- `帮我判断这个值不值得沉淀`

默认处理方式：

1. 先判断是否值得入库。
2. 值得入库时，提炼成可调用知识，不机械保存大段原文。
3. 写清楚来源、时间、为什么保存、可用于什么决策、待验证项。
4. 关联已有产品档案、项目分析、方法论或历史记录。

## 查找和回顾时

你不用知道资料在哪里，可以直接问：


```text

```


## 判断资料是否有用

优先保存这几类：

- 会影响收入、增长、产品判断或资源分配的信息。
- 能反复用于素材、投放、竞品、市场、产品分析的信息。
- 和 ROK、COD、SLG、游戏发行、买量广告直接相关的信息。
- 有明确观点、数据、案例、方法或反常信号的信息。

默认不沉淀这几类：

- 只有情绪，没有事实或判断依据。
- 只是短期八卦，未来很难调用。
- 无法确认来源，又会影响重要决策的结论。
- 版权或隐私风险过高，且无法转写成自己的判断。

## 输出标准

一次有效调用，至少应该输出：

- 结论是什么。
- 依据来自哪些知识库内容。
- 证据强度如何。
- 还缺什么数据。
- 下一步该做什么。
